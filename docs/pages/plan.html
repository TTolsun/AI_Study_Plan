Camera HAL AI 학습 계획

이번 스터디는 AI Agent 기본 활용을 익히는 교육과, Camera HAL 업무에 적용할 실전 아이템을 함께 운영하는 방식으로 진행합니다. 초반에는 공통적인 사용법과 검증 습관을 익히고, 이후에는 현업 아이템 공유와 교육을 병행합니다. 실전 적용 과정에서 나온 질문과 어려운 내용은 다음 회차 교육 주제로 연결하며, 12월부터는 AI Expert 시험 대비로 확장합니다.

운영 원칙

이번 스터디는 단순히 기능을 배우는 교육이 아니라, 실제 업무에 적용하면서 필요한 내용을 함께 쌓아가는 방식으로 운영합니다.

원칙 1

초반에는 공통 기반부터 익히기

초반에는 AI Agent 기본 사용법, 컨텍스트 설계, 검증 습관을 공통으로 익히는 데 집중합니다.

원칙 2

교육과 실전 아이템 병행

이후에는 현업 아이템을 진행하면서 필요한 내용을 함께 학습합니다. 실전 적용 과정에서 나온 질문과 이슈를 교육 주제로 연결합니다.

원칙 3

공유 기반의 유연한 운영

회차마다 아이템 진행 상황과 어려운 점을 공유하고, 공유할 내용이 없는 경우에는 AI Agent 교육 중심으로 운영합니다.

핵심은 정해진 교육을 따라가는 것이 아니라, 실제 현업 적용 과정에서 필요한 내용을 함께 보완해 나가는 운영 구조를 만드는 것입니다.

스터디에서 잡을 공통 프레임

스터디에서는 AI Agent를 단순 질의응답 도구가 아니라, 실제 업무를 보조하는 실전 도구로 활용하기 위한 공통 기준을 함께 맞춥니다.

컨텍스트 설계

AI Agent에게 무엇을 읽히고, 어떤 제약을 주고, 어떤 산출물을 기대할지를 먼저 정합니다. 코드, 로그, 설계 문서, 규칙 문서를 어떻게 조합해 넣느냐에 따라 결과 품질이 크게 달라집니다.

Rules Logs Design Docs Outputs

실전 가드레일

권한, 자동 개입 규칙, 폴더 구조, 검증, 산출물 정리 방식을 함께 잡습니다. AI에게 무엇을 읽히고 어디까지 맡길지, 결과를 어떻게 확인할지를 설계하는 감각을 익힙니다.

Permissions Hooks Verify Handoff

학습 모듈

기본 교육은 아래 흐름으로 준비하되, 실제 운영 중에는 현업 아이템에서 나온 질문과 이슈를 반영해 유연하게 조정합니다.

Module 1. AI Agent 기본 활용

입문

AI Agent를 단순 채팅 도구가 아니라, 문서를 읽고 정리하고 코드 작업을 도와주는 실무형 도구로 이해합니다.

  • AI Agent가 할 수 있는 일과 못 하는 일
  • 질문형 사용과 구현형 사용의 차이
  • 작고 되돌리기 쉬운 첫 과제 고르기

Module 2. 첫 실습과 기본 운영 방식

실습

설치보다 중요한 것은 어떤 파일을 먼저 읽히고, 어떤 기준으로 결과를 검증할지 정하는 것입니다. 초반에는 안전한 첫 사용 경험을 만드는 데 집중합니다. Camera HAL 업무 기준으로는 복잡한 동작을 바로 수정하기보다, 기존 로그나 문서를 바탕으로 흐름을 정리하고 작은 보조 작업부터 맡기는 방식으로 시작합니다.

  • 기본 규칙 파일과 작업 범위 정리
  • 테스트·검증 루틴 이해
  • 작은 예제로 plan → implement → verify 흐름 체험
  • 예시: capture request / result flow 요약, 기존 디버깅 로그 정리

Module 3. 파일 구조와 템플릿

구조화

배경 정보, 작업 파일, 결과 파일, 템플릿을 분리해 관리하고, 나중에 다시 써먹을 수 있는 구조를 만드는 방법을 익힙니다.

  • 배경 파일, 작업 파일, 결과 파일 분리
  • template과 reference의 차이 이해
  • plan, handoff, decision log 역할 정리

Module 4. 권한과 검증

가드레일

빠르게 돌리는 것보다 안전하게 반복 가능하게 만드는 것이 중요합니다. 권한 범위, 민감 경로, 검증 기준, 자동화 규칙을 함께 다룹니다.

  • 설정 범위와 팀/개인 설정 분리
  • 고위험 작업 경계 설계
  • hooks와 verify를 통한 반복 작업 자동화

Module 5. IDE 연동과 실전 워크플로우

개발 흐름

VS Code 연동, diff 확인, 계획 검토, 여러 파일 수정, handoff 정리처럼 실제 개발 흐름에서 자주 쓰게 되는 기능을 실습합니다. Camera HAL 관점에서는 단순 코드 작성보다, 여러 경로에 흩어진 변경 사항을 함께 보고 로그·설계 문서·테스트 코드까지 연결해서 검토하는 흐름을 익히는 것이 중요합니다.

  • IDE 연동과 CLI 활용 차이
  • 여러 파일 수정과 검토 흐름
  • 실전 업무에 맞는 작업 패턴 정리
  • 예시: 디버깅 로그 기반 원인 추론, Gerrit 리뷰 전 변경 요약, 레거시 C++ 코드 리팩토링 초안 작성

Module 6. Skills, MCP, 운영 자산화

확장

반복 절차는 Skill로 정리하고, 외부 시스템 연동은 MCP 같은 연결 구조로 이해합니다. 실전에서 자주 쓰는 방법을 팀 자산으로 남기는 것이 목표입니다. Camera HAL 업무에서는 반복적으로 수행하는 리뷰 체크, 테스트 보강, 문서 정리 같은 작업을 재사용 가능한 형태로 축적하는 것이 특히 중요합니다.

  • 반복 절차를 재사용 가능한 형태로 정리
  • 외부 시스템 연결 개념 이해
  • 우리 팀에 필요한 운영 자산 축적
  • 예시: Negative TC 생성 보조, patch pre-check 규칙화, 설계 문서·handoff 문서 템플릿 자산화

운영 로드맵

초반에는 AI Agent 기본 사용법을 익히고, 이후에는 교육과 현업 아이템을 병행합니다. 12월부터는 AI Expert 시험 대비를 중심으로 후반부 학습을 운영합니다.

Phase 1. 초반 운영

기본 적응

초반 운영에서는 AI Agent를 실무에 적용하기 위한 공통 기반을 맞추고, 이후 실전 아이템을 진행할 수 있도록 기본 사용 흐름과 검증 습관을 정리합니다.

  • AI Agent 기본 사용법 이해
  • 질문형 사용과 구현형 사용의 차이 익히기
  • 컨텍스트 설계, 파일 구조, 검증 방식의 기본 감각 익히기li>
  • 작고 안전한 예제로 첫 실습 진행

Phase 2. 중반 운영

교육 + 실전 병행

AI Agent 교육과 현업 아이템을 함께 진행하는 핵심 구간입니다. Camera HAL 실무에서 실제로 부딪히는 로그 분석, 테스트 보강, 리뷰 준비, 문서 정리 같은 주제를 중심으로 아이템을 진행하고 사례를 축적합니다.

  • 현업 적용 아이템 진행 및 공유
  • 어려웠던 사례, 시행착오, 좋은 활용 사례 정리
  • 공유된 내용을 바탕으로 다음 교육 주제 선정
  • 공유할 내용이 없으면 AI Agent 교육 중심으로 진행
  • 예시: HAL 로그 기반 원인 후보 추리, 테스트 코드 보강 포인트 도출, 리뷰 코멘트 초안 작성, 설계 문서 최신화

Phase 3. 후반 운영

AI Expert 대비

실전 적용 경험을 바탕으로 AI Expert 시험 대비를 운영합니다. 단순 이론 암기가 아니라, 현업에서 사용해 본 경험과 시험 개념을 연결하는 방향으로 진행합니다.

  • Python 및 AI 기초 개념 정리
  • 핵심 이론 복습과 문제 풀이
  • 현업 적용 경험과 시험 개념 연결

회차 운영 방식

매 회차는 고정된 강의형 진행보다는, 교육과 실전 공유를 함께 운영하는 방식으로 진행합니다.

기본 진행 흐름

  • AI Agent 활용 사례 또는 현업 아이템 진행 상황 공유
  • 막혔던 내용, 어려웠던 내용, 같이 볼 만한 내용 정리
  • 필요한 주제를 중심으로 교육 또는 데모 진행
  • 다음 회차까지의 실습 또는 적용 과제 정리

운영 방식의 특징

  • 공유할 내용이 있으면 실전 중심으로 진행
  • 공유할 내용이 없으면 AI Agent 교육 중심으로 진행
  • 교육 주제는 실제 현업 이슈를 반영해 유연하게 조정
  • 실습 결과와 운영 노하우는 팀 자산으로 축적
