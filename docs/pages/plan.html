학습 원칙

이번 계획은 “기능을 많이 아는 것”보다 “어떤 작업을 어떤 경로에서 시작하고, 어떤 가드레일 아래에서 검증할지”를 익히는 데 초점을 둡니다.

원칙 1

작고 되돌리기 쉬운 과제부터

처음에는 회의록 정리, 짧은 브리프, 작은 버그 수정처럼 원본이 분명하고 사람이 다시 확인할 수 있는 과제로 시작합니다.

원칙 2

프롬프트보다 파일과 구조

ABOUT-ME, TEMPLATE, OUTPUT, CLAUDE.md, plan, handoff처럼 읽을 것과 남길 것을 구조로 분리하는 습관을 먼저 익힙니다.

원칙 3

속도보다 권한·검증·하네스

permissions, hooks, review, verify 기준을 먼저 잡고, 그 뒤에 자동화와 반복 운영으로 넓혀 갑니다.

핵심은 “AI가 똑똑한가”보다 “우리 작업이 덜 끊기게 설계되었는가”입니다. 같은 도구도 작업 경로와 검증 습관에 따라 체감 품질이 크게 달라집니다.

스터디에서 잡을 공통 프레임

가이드북 내용을 기준으로, 스터디에서는 아래 네 틀을 공통 언어로 사용합니다.

작업 경로

Chat, Projects, Cowork, Claude Code처럼 어떤 책상에서 일을 시작할지 구분합니다. 질문형, 문서형, 구현형 과제를 섞지 않는 것이 첫걸음입니다.

Chat Projects Cowork Claude Code

하네스

권한, 자동 개입 규칙, 폴더 구조, 테스트, 저장 위치를 묶은 작업장입니다. AI에게 “무엇을 읽고, 어디까지 수정하고, 무엇을 확인한 뒤 끝낼지”를 설계하는 층입니다.

Permissions Hooks Verify Handoff

학습 모듈

공식 문서의 핵심 기능과 가이드북의 실무 개념을 합쳐, 실제 스터디는 아래 모듈 순서로 진행합니다.

Module 1. Claude Code 기본 감각

입문

Claude Code를 단순 채팅 도구가 아니라 코드베이스를 읽고, 파일을 수정하고, 명령을 실행하는 개발용 에이전트로 이해합니다.

  • Claude Code가 할 수 있는 일과 못 하는 일
  • 질문형 사용과 구현·검증형 사용의 차이
  • 작고 되돌리기 쉬운 첫 과제 고르기

Module 2. 첫 30분 온보딩

실습

설치 자체보다 중요한 것은, 어떤 파일을 먼저 읽히고 어디에 결과를 남길지 정하는 것입니다. Claude Code는 CLAUDE.md, test/lint/build, safety 경계를 먼저 잡아야 덜 흔들립니다.

  • 최소 CLAUDE.md 작성
  • 테스트·린트 명령 정리
  • 작은 버그를 대상으로 plan → implement → verify 루틴 체험

Module 3. 파일 구조와 템플릿

구조화

ABOUT-ME, PROJECTS, TEMPLATES, OUTPUTS 같은 폴더 역할을 분리하고, 템플릿과 브리프를 따로 관리하는 이유를 익힙니다.

  • 배경 파일, 작업 파일, 결과 파일 분리
  • template과 reference의 차이
  • plan.md, handoff.md, decision log의 역할

Module 4. 권한과 검증

가드레일

allowlist, deny 규칙, 민감 경로 차단, review split, hooks 같은 기본 가드레일을 익힙니다. 빠르게 돌리는 것보다 안전하게 반복 가능하게 만드는 것이 목표입니다.

  • settings 범위와 팀/개인 설정 분리
  • .env, 삭제, 배포 같은 고위험 경계 설계
  • hooks로 “기억나면 하자”를 자동 규칙으로 전환

Module 5. IDE 연동과 실전 워크플로우

개발 흐름

VS Code 연동, inline diff, plan review, @-mention, Git/worktree 등 실제 개발 흐름에서 체감되는 기능을 실습합니다.

  • VS Code 확장과 CLI의 역할 차이
  • 여러 파일 수정과 검토 흐름
  • worktree, review split, handoff를 활용한 병렬 작업

Module 6. Skills와 MCP

확장

반복 절차는 Skill로, 외부 시스템 연결은 MCP로, 팀 표준 배포는 Plugin 또는 팀 규칙으로 읽는 감각을 익힙니다.

  • SKILL.md 기반 반복 절차 재사용
  • MCP를 통한 외부 도구 연결 개념 이해
  • 우리 팀 업무에 필요한 연결과 아닌 연결 구분

연간 운영 로드맵

2주 1회, 90분 운영 기준으로 초반은 공통 기반 학습, 중반은 실전 적용, 후반은 각자 선택한 AI Item 실습과 AI Expert 연계로 가져갑니다.

Phase 1. 4월 ~ 5월

기본 감각

작업 경로, 하네스, 첫 30분 온보딩, 파일/폴더 구조, plan-before-edit 감각을 익힙니다.

  • Claude Code 개요와 Quickstart
  • 작은 성공 과제 선정
  • CLAUDE.md, test, lint, output 구조 만들기

Phase 2. 6월 ~ 7월

운영 구조

설정 범위, 권한 모드, hooks, verify, handoff, review split처럼 장기적으로 오래 남는 운영 습관을 붙입니다.

  • settings scope 이해
  • 권한 허용 목록 설계
  • 작업 후 검증 및 handoff 기록

Phase 3. 8월 ~ 9월

확장 기능

IDE 연동, Skills, MCP, subagent/worktree 같은 기능을 실습하고, 우리 팀에 필요한 것만 좁혀서 가져갑니다.

  • VS Code 실전 흐름
  • 반복 작업을 Skill로 정리
  • MCP와 연결 범위 판단

Phase 4. 10월 ~ 11월

현업 적용

효과적인 AI Item을 함께 정리한 뒤, 각자 하고 싶은 아이템을 선택해 자유롭게 진행합니다.

  • 개별 또는 소규모 아이템 선정
  • PoC와 중간 리뷰
  • 가이드, 템플릿, 규칙으로 팀 자산화

Phase 5. 12월 이후

확장 학습

실무 적용 경험을 바탕으로 AI Expert 시험 대비 흐름으로 자연스럽게 연결합니다.

  • Python 튜토리얼 재정리
  • CS231n / CME295 기반 핵심 개념 복습
  • 현업 적용 경험과 시험 개념 연결

회차 운영 방식

매 회차는 너무 이론적이거나 너무 산만해지지 않도록, 개념과 실제 실습을 함께 가져갑니다.

90분 기본 템플릿

  • 20분: 핵심 개념 정리
  • 25분: 실제 데모 또는 사례 리뷰
  • 35분: 직접 실습 또는 미니 과제
  • 10분: 회고, 다음 회차 준비, handoff

매 회차 산출물

  • 짧은 실습 결과 또는 예제 파일
  • 팀 공용 규칙 또는 템플릿 업데이트
  • 다음 회차를 위한 질문 / TODO / handoff
